# 요리 초보 어시스턴트 챗봇 — spec.md

## 개요

사용자가 요리명을 입력하면 필요한 재료를 먼저 안내하고, 이후 대화를 통해 단계별로 요리를 진행할 수 있도록 돕는 웹 채팅 인터페이스. Claude API의 tool use, streaming, multi-turn 역량을 관찰 가능한 형태로 시연한다. 별도의 재료 관리 탭에서 마트 영수증(mock 스캔)이나 직접 입력으로 식재료를 등록하면, 채팅에서 저장된 재료를 자동으로 활용한다.

## 범위

### 포함
- **채팅 탭**: 어시스턴트 인사 + 요리명 입력 → `get_recipe` tool 호출 → 재료 목록 표시 → 단계별 대화 진행
- 스트리밍 응답 (타이핑 효과)
- 대화 컨텍스트 유지 (세션 내, 새로고침 시 초기화)
- 없는 재료 처리: 대체 재료 또는 생략 안내
- 가상 `get_recipe` tool 시연
- **재료 탭**: 재료 목록 조회·삭제, 유통기한 D-뱃지 표시
- 재료 추가 방법 선택 (영수증 스캔 / 직접 입력)
- 영수증 스캔: 이미지 선택 → mock 파싱 결과 → 항목 선택 → 저장
- 직접 입력: 재료명·수량·단위·유통기한·메모 폼
- **재료-채팅 연동**: "냉장고 재료로 만들어줘" 입력 시 저장된 재료 자동 주입
- 재료 데이터는 localStorage에 영속 저장 (새로고침 후에도 유지)

### 제외
- 로그인·인증 (없음, 누구나 사용) — 포트폴리오용 단일 사용자 앱
- 실제 OCR API 연동 — mock으로 대체 (3-4시간 범위 초과)
- Push 알림 (유통기한 만료 전 알림) — 구현 복잡도 대비 핵심 가치 낮음
- 외부 레시피 DB 연동
- 이미지 인식 (재료 사진으로 인식하는 기능)

## 시나리오

---

### ── 채팅 탭 ──

### 1. 첫 진입 — 어시스턴트 인사
- **Given** 채팅 페이지가 열려 있음
- **When** 페이지 최초 로드
- **Then** 채팅 탭이 활성화되고, 어시스턴트 인사 메시지와 제안 칩 4개가 표시됨

성공 기준:
- [ ] 페이지 로드 → 어시스턴트 메시지가 표시됨 (제안 칩 포함)
- [ ] 사용자가 아직 아무 입력도 하지 않은 상태에서 입력창이 활성화되어 있음

---

### 2. 요리명 입력 → 재료 목록 표시 (Tool Use 시연)
- **Given** 채팅 탭이 열려 있고 어시스턴트 인사 메시지가 표시된 상태
- **When** 사용자가 "계란볶음밥 만들어줘"를 입력하고 전송
- **Then**
  - get_recipe tool 호출 중임을 나타내는 뱃지가 표시됨
  - 응답 텍스트가 글자 단위로 스트리밍되어 나타남
  - 응답에 필요한 재료 목록이 포함됨
  - 응답 마지막에 시작 확인을 유도하는 문장이 포함됨

성공 기준:
- [ ] "계란볶음밥 만들어줘" 전송 → tool 호출 뱃지가 먼저 나타남
- [ ] 응답 완료 전에 텍스트가 부분적으로 화면에 표시됨 (스트리밍)
- [ ] 응답 중 입력창과 전송 버튼이 비활성화 상태임

---

### 3. 단계별 요리 진행 — 대화형 가이드
- **Given** 재료 목록이 표시된 상태
- **When** 사용자가 "준비됐어요" 전송
- **Then** 구체적인 행동이 담긴 첫 번째 조리 단계가 표시되고, 완료 확인을 유도하는 문장으로 끝남

성공 기준:
- [ ] "준비됐어요" 전송 → 응답에 번호가 붙은 조리 행동 설명이 포함됨
- [ ] 이전 대화의 요리명·재료를 다시 묻지 않고 연속된 맥락으로 응답함

---

### 4. 없는 재료 처리
- **Given** 재료 목록이 표시된 상태
- **When** 사용자가 "버터가 없어요" 전송
- **Then** 대체 재료 또는 생략 가능 여부가 안내되고, 다음 단계 진행을 유도하는 문장으로 끝남

성공 기준:
- [ ] "[재료명]이 없어요" 전송 → 응답에 대체 재료 또는 생략 가능 여부 텍스트가 포함됨
- [ ] 안내 후 다음 단계 진행을 유도하는 문장이 포함됨

---

### 5. 중간에 다른 요리로 변경
- **Given** 한 요리의 단계별 가이드가 진행 중인 상태
- **When** 사용자가 "그냥 된장찌개로 바꿀게요" 전송
- **Then** 된장찌개의 재료 목록이 새로 표시되고, 이전 요리 맥락에서 전환됨

성공 기준:
- [ ] 다른 요리명 전송 → 응답에 새 요리("된장찌개")의 재료 텍스트가 포함됨
- [ ] 이전 요리("계란볶음밥")의 단계 안내가 이어지지 않음

---

### 6. 세션 초기화 (새로고침)
- **Given** 여러 턴의 대화가 진행된 상태
- **When** 브라우저 새로고침
- **Then** 채팅창이 초기 상태로 돌아가 어시스턴트 인사 메시지만 표시됨

성공 기준:
- [ ] 새로고침 후 → 이전 대화 메시지가 화면에 표시되지 않음
- [ ] 새로고침 후 → 어시스턴트의 첫 인사 메시지가 다시 표시됨

---

### 7. 저장된 재료를 채팅에서 자동 활용
- **Given** 재료 탭에 "계란(6개)", "대파(1단)"가 저장된 상태에서 채팅 탭으로 전환
- **When** 사용자가 "냉장고 재료로 만들어줘" 전송
- **Then** 저장된 재료 목록이 응답에 반영됨 (계란, 대파 언급 포함)

성공 기준:
- [ ] "냉장고 재료로 만들어줘" 전송 → 응답에 저장된 재료명("계란", "대파")이 포함됨
- [ ] 저장된 재료가 없는 상태에서 동일 입력 → "저장된 재료가 없다"는 안내 또는 일반 재료 추천이 표시됨

---

### 8. API 오류 처리
- **Given** 채팅창이 활성화된 상태
- **When** 메시지 전송 시 Claude API 호출이 실패함
- **Then** 오류 메시지가 채팅창에 표시되고, 입력창이 즉시 재활성화됨

성공 기준:
- [ ] API 오류 발생 → 오류 안내 메시지가 표시됨 (어시스턴트 버블과 시각적으로 구분)
- [ ] 오류 표시 후 입력창이 다시 활성화됨

---

### ── 재료 탭 ──

### 9. 재료 목록 조회
- **Given** localStorage에 저장된 재료가 있는 상태
- **When** 재료 탭 클릭
- **Then** 저장된 재료 목록이 표시되고, 각 항목에 이름·수량·유통기한·D-뱃지가 보임

성공 기준:
- [ ] 재료 탭 클릭 → 저장된 재료 항목(이름, 수량) 목록이 표시됨
- [ ] 유통기한이 있는 항목은 "D-N" 형식의 뱃지가 표시됨
- [ ] 유통기한 7일 이내 항목은 "곧 만료" 섹션에 별도로 표시됨

---

### 10. 재료 탭 — 빈 상태
- **Given** 저장된 재료가 없는 상태
- **When** 재료 탭 클릭
- **Then** 재료가 없음을 안내하는 빈 상태 메시지와 추가 유도 버튼이 표시됨

성공 기준:
- [ ] 재료 없음 → "저장된 재료가 없어요" 또는 유사한 빈 상태 안내 텍스트가 표시됨
- [ ] 빈 상태에서도 재료 추가 버튼이 활성화되어 있음

---

### 11. 영수증 스캔으로 재료 추가 (mock)
- **Given** 재료 추가 방법 선택 화면에서 "영수증 스캔" 선택
- **When** 이미지 파일을 선택함
- **Then** 미리 정의된 mock 파싱 결과(재료 목록)가 표시됨

성공 기준:
- [ ] 이미지 선택 → 재료명·수량이 채워진 파싱 결과 목록이 표시됨 (3개 이상)
- [ ] 각 항목에 체크박스가 있고, 기본 상태는 선택됨
- [ ] 유통기한 입력란이 항목별로 표시됨 (비어 있거나 자동 입력)
- [ ] 비식품 항목(예: 비닐봉지)은 기본 선택 해제 상태로 표시됨

---

### 12. 스캔 결과 선택 후 저장
- **Given** 영수증 스캔 결과 화면에서 저장할 항목이 선택된 상태
- **When** "저장" 버튼 클릭
- **Then** 선택된 항목들이 재료 목록에 추가되고, 재료 목록 화면으로 이동함

성공 기준:
- [ ] "저장" 버튼 클릭 → 재료 탭 목록에 선택된 항목 수만큼 항목이 추가됨
- [ ] 저장 후 재료 목록 화면(Screen 5)이 표시됨
- [ ] 새로고침 후에도 저장된 재료가 목록에 남아 있음 (localStorage 영속)

---

### 13. 직접 입력으로 재료 추가
- **Given** 직접 입력 폼이 표시된 상태
- **When** 재료명과 수량을 입력하고 "저장" 버튼 클릭
- **Then** 입력한 재료가 재료 목록에 추가되고, 재료 목록 화면으로 이동함

성공 기준:
- [ ] 재료명 + 수량 입력 후 저장 → 재료 목록에 해당 항목이 나타남
- [ ] 저장 후 재료 목록 화면(Screen 5)이 표시됨
- [ ] 새로고침 후에도 저장된 재료가 목록에 남아 있음

---

### 14. 직접 입력 — 필수 필드 누락
- **Given** 직접 입력 폼이 표시된 상태
- **When** 재료명 또는 수량을 비운 채 "저장" 버튼 클릭
- **Then** 저장되지 않고 누락된 필드를 안내하는 메시지가 표시됨

성공 기준:
- [ ] 재료명 비움 → "재료명을 입력하세요" 또는 유사한 안내가 해당 필드 근처에 표시됨
- [ ] 수량 비움 → 유사한 안내가 표시됨
- [ ] 안내 표시 후 현재 폼 화면에 그대로 남아 있음 (화면 이동 없음)

---

### 15. 재료 삭제
- **Given** 재료 목록에 항목이 1개 이상 있는 상태
- **When** 특정 항목의 삭제 버튼 클릭
- **Then** 해당 항목이 목록에서 사라지고, 나머지 항목은 유지됨

성공 기준:
- [ ] 삭제 버튼 클릭 → 해당 항목이 목록에서 즉시 사라짐
- [ ] 다른 항목들은 그대로 남아 있음
- [ ] 새로고침 후에도 삭제된 항목이 다시 나타나지 않음

---

## 불변 규칙

**응답 중 중복 전송 방지:**
어시스턴트가 응답 스트리밍 중인 동안은 입력창과 전송 버튼이 비활성화된다. 모든 채팅 전송 시나리오에 적용.

**재료 데이터 영속성:**
직접 입력 또는 스캔으로 저장된 재료는 브라우저 새로고침 후에도 유지된다. 삭제 액션을 직접 취하지 않는 한 사라지지 않는다.
